### 개요

- Stock 대시보드 고도화의 정의: 단순 대시보드가 아닌 사용자와 interactive. 정보의 구조화
- 주요 구조적 변화
    - Interactive: Chatbot 탑재, 그래프 Customizing (대화 내용 실시간 반영)
    - 정보 구조화: 숫자/테이블형 데이터의 벡터화는 어떻게? 또는 그래프화 필요?
    - 메모리 탑재: Auth 적용 User의 경우 History DB 저장후 소환
- 주요 Add-on (or Agent)
    - Auth
    - Chatbot
    - 숫자/테이블 데이터 Vector DB: 지표에 대한 사전 지식 주입?
    - 그래프 Custom
    - Memory (by User RLS)
    - Onboard Agent


### 추가 사항

### 2-1. 그래프별 시나리오

- 상관도: 관계도가 높은 자산을 Grouping
    - 관계 높은 자산에 대한 설명
    - 과거 경험으로 수익률이 높은 다른 자산의 흐름 → 상관 자산 추천
- 지표: RSI / MACD (틀려 있음)/ 볼린저 밴드 추가? / ATR
    - MACD: 모멘텀 관련 (12, 26) → MACD 9 EMA
    - RSI (14) (feat Bollinger): Contrarian 관련
    - Bollinger + ATR: 위험 회피 관련
    - 전체적으로 **현재** 이 전략들이 잘 맞고 있는지 Showing → 성공 확률로 보여주기
- 전략:
    - 모멘텀 vs Contrarian vs 위험회피
    - 과거 2년/1년/6개월 수익률로 보여주기 ⇒ Schematic을 잘 사용

### 2-2. 그래프별 커스텀 사항

- 상관도: 요청시 종목간 scatter plot 보여주기
- 지표: 요청시 주가와 지표간에 Overlap해서 보여주기. 정규화해서 보여주기
- 전략: 종목간 또는 전략간 성과 비교해서 보여주기

### 2-3. 기존 API 및 Front 재편

- 페이지는 홈/가격/상관/지표 시그널/전략
- 팩터와 시그널 통합(지표 시그널로 명칭): 위의 지표들에 대한 시그널 표시
- 전략 페이지는 지표 시그널에 기반해서 성과를 표시해주어야 함

### 2-4. 기존 이슈 개선 필요 사항

- 전체 페이지 모두 설명 보강 필요. 용어가 너무 어려운 경우가 많으니 초심자도 접근 가능한 용어로 변경
- 가격 및 지표 페이지에서 가격으로 인해 서로 비교가 어려운 경우 정규화를 선택 가능하게 할 것
- 상관 페이지: 자산간 교차점 버튼 클릭시 상관 scatter 도시. 챗봇 기반 사용자 커스텀도 추가
- 지표 시그널 페이지 (팩터와 통합 이후):핵심 지표들에 대해서 표시 및 시그널 표현. 챗봇 기반 사용자 커스텀도 추가
- 전략 페이지: 기간별 (6M, 1Y등) 성과 선택 가능하도록 변화
