## CC와 논의 항목

### 1.각 서비스 페이지(상관도, 지표, 전략)의 구체적 서비스 내용
- 상관도
    - 유사 그룹 제공
    - 종목 선택시 유사 자산 제공 (유사도 순서로)
    - 유사 자산간 갭 메우기 과거 이력 제공
- 지표
    - 지표별매수/매도 시점 제공. 각 지표별 기준 숫자 명확해야 (사용자-CC 논의 중요)
    - 지표별 매수/매도 성공률 제공 (기준: 5일후 가격 상승/하락, 확률 및 변화율)
    - 각 지표의 설명도(성공률) 비교 제공
- 전략
    - 전략별  사전 설명 제공 (ex. 모멘텀 전략은 달리는 말에 타는 전략… 지표는 MACD를 이용…)
    - 전략별 1억원 시드의 시점별 자산 변화 (시각적) 제공
    - 전략별 / 시점별 이벤트(수익, 위험 양쪽 관점)를 스토리텔링하는 UX 제공
### 2.Phase C(각 서비스 고도화) 및 Phase D(그래프 커스텀) Phase 통합 여부
- 그래프 커스텀 구현 항목
    - 상관도 관심 종목 설정
    - 지표 (or 가격) 멀티 설정
    - 지표 (or 가격 ) 단위 설정
    - 지표 (or 가격) 정규화 설정
      - 전략 기간 설정
### 3. 각 서비스의 결과를 보여주는 방식
- 사용자 UX Flow: 피싱 질문이 있어야 함. 넛지 & 선택 형태로 가야 함
- 응답 방식: 1) 템플릿 기반, 2) llm 기반, 3) 하이브리드?
    - 하이브리드 결정. 피싱 질문에 의해 카테고리중 선택 → 템플릿 응답 → 카테고리에 미해당의 경우 일반 응답으로 넘기기 → 페이지별 컨텍스트에서 검색 → 검색 결과 llm으로 해석하여 응답
- 구현 항목: 3가지가 필요. 1) 페이지별 컨텍스트 전부 (구조화), 2) 템플릿 형태 응답, 3) llm 응답 (w/템플릿 + 컨텍스트 조합)
