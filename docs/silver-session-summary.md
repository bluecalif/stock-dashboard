## 1. 목적
- `docs/silver-draft.md`를 기반으로 Silver 업그레이드 계획을 implementation-ready 수준으로 구체화하기 위한 중간 정리 문서
- 이 문서는 현재 세션에서 확정된 사항과 아직 논의가 필요한 사항을 compact 이후 다시 이어가기 쉽게 정리한다

## 2. 핵심 사용자와 제품 목표
- `rev1`의 1차 핵심 사용자는 `초보 투자자`
- 초보 투자자가 `여러 상품에 대한 적립식 투자의 기대 수익과 위험`을 쉽게 이해할 수 있어야 함
- 동시에 향후에는 `리밸런싱`, `시그널`, `기존 투자자용 전략 검토`로 확장 가능한 기반이 되어야 함

## 5. 적립식 모델 기본 규칙 확정
- 적립 주기: `매월`
- 적립 시점 기본: `매월 첫 거래일`
- 기본 적립 금액: `월 100만원`
- 투자 기간: `기본 10년`, 사용자 조정 가능
- 화면 기본 기간도 `10년`
- 핵심 시뮬레이션 가정:
  `현재가 10년 전 오늘이라고 가정`하고,
  `과거 10년 동안 실제로 일어난 가격 흐름이 앞으로 10년 동안 그대로 반복된다`는
  replay 방식으로 보여줌
- 배당/이자 성격 자산은 `재투자 고정`
- 세금/수수료/슬리피지: `rev1 제외`
- 통화 표시: `원화`, `현지통화` 둘 다 지원
- 기본 표시 통화: `원화`
- 환율 기준: `현재 기준 환율`
- 적립 금액 입력 방식:
  `프리셋 기반`으로 두고, 기본값은 `월 100만원`

## 6. 시간/조건 기반 전략 비교 확정

### 6.1. 전략의 성격
- 이 영역은 복합 전략 1개가 아니라 `서로 독립적인 2개 전략` 비교
- 비교 대상은 `단일 자산` 기준
- 대상 자산은 `QQQ`, `SPY`, `KOSPI200`

### 6.2. 전략 A: 고가 매도 후 저가 재매수
- 급등 조건: `3개월 내 20% 상승`
- 조건 만족 시 `현재 전체 보유수량의 30% 매도`
- 재매수 금액은 `매도해서 생긴 현금 전액`
- 재매수는 `1년 내 급락 조건` 만족 시 실행
- 급락 조건이 1년 내 없으면 `12월 마지막 거래일`에 재매수
- 같은 해에 조건이 여러 번 발생해도 `1회만 실행`

### 6.3. 전략 B: 70% 정기매수 + 30% 대기매수
- 매년 적립 예정 금액 중 `70%만 정기 매수`
- 나머지 `30%는 급락 조건 대기 자금`
- 급락 조건: `정확히 20거래일 전 대비 10% 하락`
- 급락 조건 만족 시 대기 자금 매수
- 조건이 없으면 `12월 마지막 거래일`에 매수
- 이 전략은 적립식의 일부이며, 전략 A와 별도로 비교

### 6.4. 비교 지표
- `누적수익률`
- `연환산수익률`
- `최고 하락률`

## 7. 최고 하락률 정의 논의 결과
- 대표 위험지표 후보 중 `연도별 MDD 중 최악의 값` 방향으로 합의
- 단, 구현 문구는 더 정확히 써야 함
- 사용자가 중요하게 본 기준:
  적립식 `누적 자산/누적 수익률 곡선`을 기준으로,
  예를 들어 `40% 수익 상태에서 20% 수익 상태로 내려오는` 식의 체감 가능한 감소를 반영해야 함
- 연도별 MDD는 `그 해 안의 고점 이후 저점` 기준이어야 한다는 점에 합의
- 이 항목은 문서 작성 시 정의를 더 엄밀히 다듬어야 함

## 8. 포트폴리오 비교 확정

### 8.1. 기능의 정확한 성격
- 핵심은 `자산 리밸런싱` 자체보다 `단일 자산 100% 적립식`과 `고정 포트폴리오 적립식`의 비교
- 포트폴리오 내부 리밸런싱은 주기능이 아니라 `포트폴리오 유지 규칙`

### 8.2. 포트폴리오 기본 규칙
- 기본 비중: `주식/ETF 60`, `TLT 20`, `BTC 20`
- 매월 적립금은 `목표비중대로 바로 배분`
- 연간 리밸런싱 시점: `매년 마지막 거래일`
- 리밸런싱 기준: `보유자산 total 기준으로 목표비중 복원`

### 8.3. 주식/ETF 60% 슬롯 후보
- `QQQ`
- `KOSPI200`
- `삼성전자/하이닉스 1:1`
- `엔비디아/구글/테슬라 1:1:1`

### 8.4. 기본 포트폴리오 preset

- `삼성전자/하이닉스 + TLT + BTC`
- `엔비디아/구글/테슬라 + TLT + BTC`

### 8.5. WBI 사용 범위
- `WBI`는 `단일 자산 비교용 benchmark`
- 포트폴리오 구성 자산으로는 사용하지 않음

## 9. 시그널 기능 방향 확정
- rev1에서는 `간략 버전`으로 포함
- 지표는 `RSI`, `MACD`, `ATR` 모두 볼 수 있어야 함
- 기본 표현 방식: `신호 요약 + 선택 오버레이형`
- 기본 차트에는 `중요 신호만 제한적으로 표시`
- 상태 문구는 `매수/매도 추천형`이 아니라 `설명형`
- 적립식 메인 각 탭 안에는 `지표 요약 카드`를 둘 수 있음
- 상세는 별도 페이지로 남기되,
  `각 지표의 일부 핵심 시그널 포인트에 hover/overlay 설명이 뜨는 수준`으로 간략화
- 기존의 `정확도/백테스트 중심 지표 페이지`를 그대로 유지하는 방향은 아님
- 대상 자산:
  `QQQ`, `SPY`, `KOSPI200`, `엔비디아`, `구글`, `테슬라`, `삼성전자`, `하이닉스`

## 10. 현재 프론트엔드 구조 확인 결과

### 10.1. 실제 라우트/페이지
- `/` → `DashboardPage`
- `/prices` → `PricePage`
- `/correlation` → `CorrelationPage`
- `/indicators` → `IndicatorSignalPage`
- `/strategy` → `StrategyPage`
- `/factors`, `/signals`는 실제로 `/indicators`로 리다이렉트

### 10.2. 현재 정보구조 해석
- `Dashboard`: 요약 허브
- `Price`: 가격/정규화 수익률 비교
- `Indicators`: 지표/시그널/정확도
- `Strategy`: 기존 전략 백테스트
- `Correlation`: 상관관계/스프레드 분석

### 10.2.A. 현재 코드 기준 실제 역할 메모
- `DashboardPage`:
  자산 카드 + 최근 백테스트 요약 허브
- `PricePage`:
  다중 자산 가격/정규화 수익률 차트 비교 페이지
- `IndicatorSignalPage`:
  RSI/MACD/ATR 시그널, 오버레이, 정확도 비교 페이지
- `StrategyPage`:
  MACD/RSI/ATR 기반 기존 단일자산 전략 백테스트 페이지
- `CorrelationPage`:
  상관관계/스프레드 분석 페이지
- rev1 작성 시 위 역할을 그대로 확장하는 것이 아니라,
  `적립식 메인 중심 정보구조`로 재해석해야 함

### 10.2.B. 현재 코드 기준 자산/기능 제약 메모
- 현재 코드/수집/백테스트는 대체로 기존 자산군
  `KS200`, `005930`, `000660`, `SOXL`, `BTC`, `GC=F`, `SI=F`
  를 중심으로 짜여 있음
- rev1 문서의 핵심 자산군
  `QQQ`, `SPY`, `SCHD`, `JEPY`, `TLT`, `NVDA`, `GOOGL`, `TSLA`, `WBI`
  는 다음 세션의 rev1 문서 작성 시
  `신규 지원/매핑/수집 확장 필요 자산`으로 전제해야 함
- 즉, 다음 세션에서 rev1 문서를 쓸 때
  `기존 데이터 파이프라인이 이 자산을 이미 지원한다`고 가정하면 안 됨

### 10.3. rev1 구조 방향성 합의
- `rev1`은 `기존 관련 페이지를 통합 재구성`하는 방향
- `적립식 비교 메인`을 첫 화면이자 주력 화면으로 사용
- 기존 `DashboardPage`는 `노출하지 않음`
- 기존 `PricePage`, `CorrelationPage`는 rev1 메뉴에서 `노출하지 않음`
- 기존 `PricePage`, `CorrelationPage`는 즉시 삭제하지 않고 `레거시 내부 유지 대상`으로 기록
- 기존 `StrategyPage`는 `완전 Drop` 대상
- 기존 `IndicatorSignalPage`는 삭제하지 않고 `기능 변경 후 재구성` 대상으로 유지
- 적립식 메인 안에서 핵심 비교축은 우선 `탭`으로 구분
- 탭 내부를 `서브 화면`으로 더 나눌지, `카드형 섹션`으로 유지할지는 실제 UI를 보며 후속 결정

## 11. rev1 UI/IA 확정본

### 11.1. 메인 메뉴 노출 정책
- 메인 주력 메뉴는 `적립식 비교`
- `IndicatorSignalPage` 계열은 `보조 상세 페이지`로 유지 가능
- 기존 `DashboardPage`, `PricePage`, `CorrelationPage`는
  `Silver 제품 마지막 단계에서 Option 검토 대상`으로 기록
- 기존 `StrategyPage`는 `완전 제거`
- 따라서 rev1 기준 핵심 사용자 플로우는
  `적립식 메인 진입 -> 탭별 비교 -> 필요 시 지표 상세 확인`

### 11.2. 적립식 메인 탭 구조
- 탭 순서 고정:
  `단일 자산 적립식 비교` ->
  `단일 자산 적립식 vs 시간/조건 전략 비교` ->
  `단일 자산 적립식 vs 포트폴리오 적립식 비교`
- 위 3축은 우선 `한 화면 안의 탭`으로 고정
- 각 탭 내부를 `서브 화면`으로 더 나눌지, `카드/섹션형`으로 둘지는
  실제 UI 검토 후 최종 확정

### 11.3. 탭별 기본 비교 조합
- 탭 A 기본 비교 자산:
  `QQQ`, `SPY`, `KOSPI200`, `SCHD`, `JEPY`, `WBI`
- 탭 B 기본 선택 자산:
  `QQQ`
- 탭 B 허용 자산:
  `QQQ`, `SPY`, `KOSPI200`만
- 탭 C 기본 포트폴리오:
  `QQQ + TLT + BTC`
- 탭 C는 `프리셋 only`

### 11.4. 사용자 기본 입력값
- 기본 기간: `10년`
- 기본 적립금 프리셋: `100만원`
- 적립금 입력은 우선 `프리셋 방식`
- 포트폴리오 비중 변경은 rev1에서 직접 편집 없이 `프리셋 only`

### 11.5. 위험 지표 사용자 문구
- 최고 하락률(MDD)의 사용자 노출 문구는
  `연도 기준 최대 손실폭`으로 고정

### 11.6. rev1 문서 톤
- 다음 세션에서 작성할 `docs/draft-rev1.md`는

## 12. 다음 세션에서 재탐색 없이 바로 사용할 구현 컨텍스트

### 12.1. 코드 구조를 다시 확인할 필요가 없는 확정 정보
- 현재 프론트 라우트는
  `/`, `/prices`, `/correlation`, `/indicators`, `/strategy`
  구조이며,
  `/factors`, `/signals`는 `/indicators`로 리다이렉트
- rev1에서는 위 구조를 그대로 노출하는 것이 목적이 아니라,
  `적립식 메인 1개 + 지표 상세 보조 구조`로 재편하는 것이 목적
- `PricePage`의 다중 자산 가격/정규화 수익률 비교 기능은
  적립식 메인 탭 A의 차트 비교 로직 설계 참고용으로 재사용 가능
- `IndicatorSignalPage`의 RSI/MACD/ATR 관련 기능은
  메인 탭 내 요약 카드와 별도 상세 페이지의 기반으로 재사용 가능
- `StrategyPage`의 기존 MACD/RSI/ATR 전략 백테스트는
  rev1의 조건 전략 비교와 개념이 다르므로
  직접 계승 대상이 아니라 `폐기/참고용`
- `CorrelationPage`는 rev1 주 사용자 플로우에서는 제외

### 12.2. rev1 문서에 반드시 반영해야 할 제품 방향
- rev1의 중심은 `초보 투자자용 적립식 비교`
- 기존 대시보드형 정보 나열보다
  `무엇을 비교해야 하는지`를 명확히 안내하는 구조가 우선
- 메인 첫 화면은 기존 요약 허브가 아니라
  바로 적립식 비교를 시작하는 화면이어야 함
- 기존 `Strategy`라는 단어는
  과거의 기술적 지표 기반 백테스트 의미와 혼동될 수 있으므로
  rev1 문서에서는
  `시간/조건 전략 비교` 또는 `조건 전략 비교`
  같은 표현으로 분리해서 써야 함

- `docs/draft-rev1.md`에서 가장 먼저

  2) 사용자 플로우
  3) 메뉴 구조
  4) 메인 탭 구조
  5) 탭별 기본값/허용 자산
  6) 주요 차트/표/설명 카드
  7) 레거시 처리 방침
  순으로 고정하는 것이 좋음
- 특히 `Dashboard`, `Price`, `Correlation`은
  `Silver 마지막 단계 Option 검토 항목`으로 명시하고,
  `Strategy`는 option이 아니라 `drop`으로 명확히 적어야 함
- `WBI`는 실자산이 아닌 가상 benchmark라는 점을
  탭 A 설명과 데이터 정의 섹션 모두에 중복 명시하는 것이 안전

## 13. 현재 남아 있는 핵심 쟁점

### 13.1. 탭 내부 레이아웃 방식
- 메인 탭 3개는 확정
- 다만 각 탭 내부를
  `카드형 한 화면`으로 둘지,
  `서브 화면/스텝형`으로 나눌지는
  실제 UI를 보며 확정해야 함

### 13.2. 최고 하락률 정의의 문서화
- `연도별 MDD 중 최악의 값` 방향은 합의
- 다만 실제 문서에서는
  `적립식 누적 자산가치 곡선 기준`,
  `연도 내 고점 이후 저점`,
  `사용자에게 보여줄 하락 폭 표현`
  을 더 엄밀히 정리해야 함
