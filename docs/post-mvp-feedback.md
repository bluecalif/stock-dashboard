## Feedback on the Phase C/D/E implementation

### 1. Phase C

- Overall: 각 그래프들에 대한 설명이 필요. 종목 코드가 아닌 종목명으로 표시
- 상관계수 분포: 삭제
- scatter plot: 히트맵의 교차점 블록을 클릭시 두 종목간 scatter plot 표시 (오른쪽)
- 스프레드
  - 그룹내 3종목인 경우 2종목을 선택할 수 있어야 (현재는 2종목이 랜덤으로 연결되어 조절이 안됨)
  - 스프레드 및 Z-score의 계산 정의가 무엇인가? 결국은 유사 종목이 갭이 벌어지면 얼마만큼 빨리 좁혀지는지 모니터링이 되어야 함. 이 관점에서 다시 리뷰
  - Z-score가 아닌 가격으로 비교해서 보여줘야. 계산을 Z-score로 해서 발산 수렴을 detect는 하지만, 가격 그래프상 갭이 벌어졌다가 가까워지는 정도(또는 기간)을 보여줘야
- 채팅
  - 질문후 스트림이 시작되기 전 시간이 오래 걸림. 대기중 무얼하고 있는지 표시해주는 UX 제공
  - 현재 어떤 질문은 템플릿으로 어떤 것은 llm으로 하고 있나? 넛지 질문은 템플릿으로 해야 하는 것 아닌지 검토
  - llm 사용시 컨텍스트로 어떤 것이 들어가나?
- 채팅 추가 개선
  - 전제: 각 페이지별로 채팅 UX 차이. 맨 위에 해당 페이지 넛지 질문 리스트
  - UX Flow: 사용자 질문 -> 1차 질문 Classification (상관관계 페이지 해당 or not) -> 분류에 따른 답변 또는 페이지 이동 추천
  - 상관 페이지 해당: 해당 Structured_Output 형식에 맞춰 llm 답변 출력. 
  - 상관 페이지 Not: 가격 / 지표시그널 / 전략 각 페이지로 이동 제안 및 실행. 각 페이지에 해당하지 않는 경우 추가 질문으로 사용자의 관심 파악하여 각 페이지로 이동 제안 및 실행
  - 이슈: 질문 Classification을 llm으로 할지 vs 벡터(or 하이브리드) 검색으로 할지 

### 2. Phase D

- Overall
  - 이 페이지의 취지가 각 지표별로 시그널을 띄우고, 그에 의한 예측 결과를 확인하는 것
  - 탭별: 지표현황 + 시그널타임라인 합치기 => 시그널로 지칭할 것, 성공률은 그대로 두기
  - 모멘텀/추세추종/평균회귀를 모든 탭에서 배제하기, 순수하게 RSI, MACD, ATR(+vol)로 시그널 및 성공률 보여주는 UX 제공
- 시그널 탭
  - 각 지표들의 시그널을 보여줄 것. 시그널 + 가격 오버레이에서만 보여주기. 시그널 지점을 Y축 방향 점선으로 알기 쉽게 표시
  - UX는 그래프를 왼쪽 3/4정도까지 보여주고, 오른쪽 1/4 부분에 설명 표시
  - **추가 오류**: 그래프는 오버레이 (지표와 가격)여야 함. 현재는 가격만 표시
- 성공률 탭
  - 각 지표들의 시그널에 의한 매매 성공률을 보여주기. 시그널 탭과 동일하나 그래프 3/4 + 설명(표) 1/4 레이아웃
  - 먼저 그래프에서 (시그널탭과 동일) 시그널 이후 매매가 성공/실패를 표시할 것. 성공시 수익률도 표시
  - 오른쪽 표에서는 각 거래별 성공실패/수익률 표시할 것
- 지표/가격 오버레이
  - 가격 정규화 적용시 0으로 튀는 포인트들이 있음. 가격 데이터가 없는 날짜 처리 주의할 것
- RSI
- MACD: **MACD와 시그널 (MACD의 9EMA, 주가의 EMA가 아님)**
- ATR(+vol): 시그널탭은 위험도 높은 구간만 표시. 성공률탭은 성공/실패만 표시

**추가 개선 Point**

- 시그널 탭
    - 지표에 대한 설명 및 판정 기준 명확/간결하게 표시할 것
    - 시그널 frequency 제어: 시그널이 전체적으로 잦은 경우 방지 → T+3일 이후에 새로 발생한 경우만 카운트 할 것
    - RSI
        - 매도/매수 신호만이 아닌 각 신호 해제도 표시 (70이하면 매도 해제 등)
        - 시그널 frequency 제어 적용 (≥T+3)
    - MACD
        - macd_signal이 25-10-1 이전에 안보임
        - 시그널 frequency 제어 적용 (≥T+3)
    - ATR+변동성
        - 변동성 값 스케일이 너무 작음.
        - ATR과 변동성 각각 시그널 기준 명확히
- 성공률 탭
    - 기간: 시그널 탭과 동일하게 선택
    - 지표별 요약
        - 예측 기간별 데이터와 매칭이 안됨
        - 성공률 기준: 예를 들면 시그널 T+5일 상승 or 하강으로 성공/실패 판정
        - 수익률 기준: 시그널 T+5일 수익률들에 대한 평균
