# Stock Dashboard Silver rev1 명세 초안

## 1. 목적
- Bronze 버전은 이미 구현 완료되어 운영 중인 기준 제품이다.
- Silver rev1의 목적은 Bronze의 기존 페이지 구조를 단순 확장하는 것이 아니라, `초보 투자자용 적립식 비교 경험` 중심으로 제품 구조를 재정의하는 것이다.
- rev1 문서는 구현 착수용 명세 문서로 사용한다.

## 1.1. 세션 고정 결정사항
이 섹션의 항목은 이미 사용자 확인이 끝난 결정사항으로 간주하며, 이후 세션에서 다시 질문하지 않는다.

| 항목 | 확정 내용 |
|---|---|
| 탭 A 기본 노출 | 기본 3개 자산 먼저 노출 |
| 공통 사용자 입력값 | 기간, 월 적립금, 자산 선택 |
| 기간 입력 방식 | `3년 / 5년 / 10년` 프리셋 |
| 월 적립금 프리셋 | `30만 / 50만 / 100만 / 200만 / 300만` |
| 탭 B 전략 A 재매수 급락 조건 | `20거래일 전 대비 10% 하락` |
| 탭 B 이벤트 순서 | `정기 적립 후 조건 실행` |
| 탭 C 리밸런싱 | 기존 보유분 포함 실제 매도/매수 |
| 탭 C 수량 규칙 | 소수 단위 매수 허용 |
| 환율 계산 방식 | 매 날짜 환율 사용 |
| 신호 기능 최소 범위 | `요약 카드 + 상세 페이지` |
| 출시 기준 | 문서상 정의한 전체 자산군 지원 완료 |
| KPI 최상위 4개 | `최종 자산`, `총 수익률`, `연환산 수익률`, `연도 기준 최대 손실폭` |
| 레거시 페이지 처리 | `Dashboard / Price / Correlation`은 Silver에서 리디렉션 또는 차단 |

## 2. 제품 목표
- 사용자가 여러 자산의 적립식 투자 결과를 직관적으로 비교할 수 있어야 한다.
- 사용자가 기대수익과 위험을 같은 화면에서 이해할 수 있어야 한다.
- 이후 확장 가능한 기반을 남겨야 한다.
  - 시간/조건 전략 비교
  - 포트폴리오 적립식 비교
  - 신호 요약 및 시각화

## 3. 핵심 사용자
- 1차 핵심 사용자는 `초보 투자자`다.
- 이 사용자는 복잡한 기술적 백테스트보다 다음 질문에 답을 얻는 것이 중요하다.
  - 어떤 자산에 적립식으로 투자하면 결과가 어떻게 달라지는가
  - 수익률이 아니라 체감 손실 폭은 어느 정도인가
  - 단순 적립식 외의 조건 전략이나 포트폴리오가 얼마나 다른 결과를 내는가

## 4. 제품 범위
### 4.1. rev1 필수 범위
- 단일 자산 적립식 비교
- 단일 자산 적립식 vs 시간/조건 전략 비교
- 단일 자산 적립식 vs 포트폴리오 적립식 비교

### 4.2. rev1 간략 포함 범위
- RSI, MACD, ATR 기반 신호 요약
- 차트 위 제한적 overlay / hover 설명

### 4.3. rev1 제외 범위
- 세금
- 수수료
- 슬리피지
- 사용자가 직접 포트폴리오 비중을 자유 편집하는 기능
- 기존 StrategyPage 개념의 기술적 전략 백테스트 메인 노출

## 5. 정보 구조
### 5.1. 메인 구조
- rev1의 메인 진입은 `적립식 비교` 단일 메뉴로 둔다.
- 첫 화면은 기존 Dashboard형 요약 허브가 아니라, 즉시 비교를 시작하는 화면이어야 한다.
- 기존 `Strategy`라는 명칭은 사용하지 않는다.
  - 문서와 UI 모두 `시간/조건 전략 비교` 또는 `조건 전략 비교`로 표기한다.

### 5.2. 메인 탭 구조
- 탭 A: `단일 자산 적립식 비교`
- 탭 B: `단일 자산 적립식 vs 시간/조건 전략 비교`
- 탭 C: `단일 자산 적립식 vs 포트폴리오 적립식 비교`

### 5.3. 보조 상세 구조
- 지표/신호 상세는 별도 보조 페이지로 유지 가능하다.
- 단, 메인 탐색 구조의 중심은 아니다.

## 6. 화면 구조
### 6.1. 디자인 방향
- `docs/UX-design-ref.JPG`는 시각 레퍼런스로만 사용한다.
- 다음 요소는 적극 차용한다.
  - 다크 톤 기반 시각 시스템
  - 상단 요약 카드
  - 메인 차트 1개 중심 배치
  - 보조 인사이트 카드
  - 높은 대비와 명확한 시선 흐름
- 다음 요소는 직접 차용하지 않는다.
  - 관리자형 좌측 사이드바 IA
  - sales / payment / report 문맥
  - 멀티 제품 대시보드 구조

### 6.2. 탭 내부 레이아웃 원칙
- 메인 탭 3개는 고정한다.
- 각 탭 내부는 우선 `한 화면 안의 카드형 섹션` 기준으로 설계한다.
- 다만 실제 구현 시 복잡도가 높은 탭은 `step형 흐름`을 예외적으로 허용할 수 있다.
- 이 예외 허용 여부는 실제 페이지 시안을 구성한 뒤 사용자가 보고 최종 결정한다.
- 1차 구현 기준은 다음 순서를 따른다.
  1. 상단 입력/프리셋 영역
  2. 핵심 비교 차트 영역
  3. 핵심 수치 카드 영역
  4. 해석 문구 / 위험 설명 영역
  5. 필요 시 지표 요약 카드

- `QQQ`
- `SPY`
- `KOSPI200`
- `SCHD`
- `JEPY`
- `TLT`
- `비트코인`
- `엔비디아`
- `구글`
- `테슬라`
- `삼성전자`
- `하이닉스`

### 7.2. 자산 정의
- `WBI`는 실자산이 아니라 `워런 버핏 연 20% 복리 가정`의 가상 benchmark다.
- `M7`은 rev1에서 `엔비디아`, `구글`, `테슬라` 3종만 포함한다.
- `SEC/SKH`는 `삼성전자/하이닉스 1:1` 조합을 의미한다.
- `KOSPI200`은 ETF가 아니라 `지수 자체` 기준이다.

### 7.3. 기존 자산 처리
- 현재 코드상 기존 자산 중 겹치는 것은 유지한다.
- `SOXL`, `금`, `은`은 삭제하지 않고 기능 호환용으로 유지한다.
- 다만 rev1 핵심 비교 대상에는 포함하지 않는다.

### 7.4. 구현 전제
- 현재 repo의 기본 지원 자산은 `KS200`, `005930`, `000660`, `SOXL`, `BTC`, `GC=F`, `SI=F` 중심이다.
- 따라서 rev1 구현 시 아래 항목은 신규 확장 대상으로 본다.
  - `QQQ`, `SPY`, `SCHD`, `JEPY`, `TLT`, `NVDA`, `GOOGL`, `TSLA`
- 내부 매핑 전제는 다음과 같다.
  - `KOSPI200 -> KS200`
  - `삼성전자 -> 005930`
  - `하이닉스 -> 000660`
  - `비트코인 -> BTC`
- `WBI`는 수집 대상이 아니라 계산식 기반 synthetic benchmark로 구현한다.

## 8. 탭 A: 단일 자산 적립식 비교
### 8.1. 목적
- 여러 자산에 같은 적립식 규칙을 적용했을 때 결과가 어떻게 달라지는지 보여준다.

### 8.2. 기본 비교 자산
- 기본 첫 노출은 3개 자산이다.
  - `QQQ`
  - `SPY`
  - `KOSPI200`
- 확장/토글 시 아래 자산을 추가 비교 대상으로 노출할 수 있다.
  - `SCHD`
  - `JEPY`
  - `WBI`

### 8.3. 기본 규칙
- 적립 주기: `매월`
- 적립 시점 기본값: `매월 첫 거래일`
- 월 적립금 프리셋: `30만 / 50만 / 100만 / 200만 / 300만`
- 기본 적립금: `월 100만원`
- 기간 프리셋: `3년 / 5년 / 10년`
- 기본 비교 기간: `10년`
- 배당/이자 성격 자산은 `재투자 고정`
- 통화 표시는 `원화`, `현지통화` 둘 다 지원
- 기본 표시 통화는 `원화`
- 환율 기준은 `매 날짜 환율`
- 적립금 입력은 `프리셋 방식`으로 시작한다.

### 8.4. 시뮬레이션 가정
- 표현상 `현재가 10년 전 오늘이라고 가정`한다.
- 과거 10년 동안 실제로 발생한 가격 흐름이 앞으로의 10년에도 반복된다는 replay 방식으로 보여준다.
- 평균수익률 단순 적용 방식은 사용하지 않는다.

### 8.5. 주요 출력
- 누적 자산가치 곡선
- 누적 수익률
- 연환산 수익률
- 연도 기준 최대 손실폭
- 최종 자산
- 총 투자원금 대비 결과

### 8.6. 사용자 설명
- `WBI`는 실자산이 아니라 가상 benchmark임을 탭 설명과 데이터 정의 모두에서 반복 명시한다.

## 9. 탭 B: 단일 자산 적립식 vs 시간/조건 전략 비교
### 9.1. 목적
- 단순 적립식과 조건 전략 2개를 같은 자산 기준으로 비교한다.

### 9.2. 허용 자산
- `QQQ`
- `SPY`
- `KOSPI200`

### 9.3. 비교 대상 성격
- 이 영역은 복합 전략 1개가 아니라, 서로 독립적인 2개 전략 비교다.
- 비교 기준선은 항상 `단일 자산 100% 적립식`이다.

### 9.4. 전략 A
- 이름: `고가 매도 후 저가 재매수`
- 급등 조건: `3개월 내 20% 상승`
- 조건 충족 시 현재 전체 보유수량의 `30% 매도`
- 재매수 금액은 `매도로 생긴 현금 전액`
- 재매수는 `1년 내 급락 조건` 충족 시 실행
- 급락 조건은 `20거래일 전 대비 10% 하락`
- 해당 기간 내 조건이 없으면 `12월 마지막 거래일`에 재매수
- 같은 해에 조건이 여러 번 발생해도 `1회만 실행`
- 정기 적립일과 조건 발생일이 겹치면 `정기 적립 후 조건 실행` 순서로 처리한다.

### 9.5. 전략 B
- 이름: `70% 정기매수 + 30% 대기매수`
- 매년 적립 예정 금액 중 `70%만 정기 매수`
- 나머지 `30%는 급락 조건 대기 자금`
- 급락 조건: `정확히 20거래일 전 대비 10% 하락`
- 조건 충족 시 대기 자금 매수
- 조건이 없으면 `12월 마지막 거래일`에 매수
- 정기 적립일과 조건 발생일이 겹치면 `정기 적립 후 조건 실행` 순서로 처리한다.

### 9.6. 주요 출력
- 누적 수익률
- 연환산 수익률
- 연도 기준 최대 손실폭
- 기준 적립식 대비 차이

## 10. 탭 C: 단일 자산 적립식 vs 포트폴리오 적립식 비교
### 10.1. 목적
- 단일 자산 100% 적립식과 고정 포트폴리오 적립식의 차이를 비교한다.

### 10.2. 포트폴리오 성격
- 핵심은 자산 리밸런싱 자체가 아니라 `고정 포트폴리오 적립식` 비교다.
- 포트폴리오 내부 리밸런싱은 주기 기능이 아니라 `포트폴리오 유지 규칙`이다.

### 10.3. 기본 규칙
- 기본 비중: `주식/ETF 60`, `TLT 20`, `BTC 20`
- 매월 적립금은 목표 비중대로 바로 배분
- 연간 리밸런싱 시점: `매년 마지막 거래일`
- 리밸런싱 기준: `보유자산 total 기준으로 목표비중 복원`
- 리밸런싱은 기존 보유분에 대한 실제 매도/매수를 포함한다.
- rev1에서는 `preset only`로 제공한다.
- 사용자가 비중을 직접 편집하는 기능은 넣지 않는다.
- 수량 계산은 소수 단위 매수를 허용한다.

### 10.4. 주식/ETF 60% 슬롯 후보
- `QQQ`
- `KOSPI200`
- `삼성전자/하이닉스 1:1`
- `엔비디아/구글/테슬라 1:1:1`

### 10.5. 기본 preset
- `QQQ + TLT + BTC`
- `KOSPI200 + TLT + BTC`
- `삼성전자/하이닉스 + TLT + BTC`
- `엔비디아/구글/테슬라 + TLT + BTC`

### 10.6. WBI 사용 범위
- `WBI`는 포트폴리오 구성 자산으로 사용하지 않는다.
- `단일 자산 비교용 benchmark`로만 사용한다.

## 11. 위험 지표 정의
### 11.1. rev1 기준 위험 지표
- 대표 위험 지표는 `연도별 MDD 중 최악의 값`으로 둔다.

### 11.2. 문서 표현 규칙
- 사용자 노출 문구는 `연도 기준 최대 손실폭`으로 고정한다.
- 구현 문서에는 다음 기준을 함께 명시해야 한다.
  - 적립식 누적 자산가치 곡선 기준
  - 해당 연도 안의 고점 이후 저점
  - 사용자가 체감하는 손실 감소를 반영하는 표현

## 12. 신호 기능 방향
### 12.1. 범위
- rev1에서는 간략 버전으로 포함한다.
- 지표는 `RSI`, `MACD`, `ATR`를 모두 볼 수 있어야 한다.

### 12.2. 표현 방식
- 기본 표현은 `신호 요약 + 선택 overlay형`
- 메인 차트에는 중요한 신호만 제한적으로 표시
- 상태 문구는 `매수/매도 추천형`이 아니라 `설명형`
- 각 탭 안에 `지표 요약 카드`를 둘 수 있다.
- 상세는 별도 페이지로 남기되, hover/overlay 설명 중심의 가벼운 구조로 단순화한다.
- rev1 필수 범위는 `요약 카드 + 상세 페이지`다.
- 상세 페이지의 IA, 레이아웃, 노출 시그널 수는 후속 논의로 확정한다.

- `하이닉스`

## 13. 사용자 입력 기본값
- 기간 프리셋: `3년 / 5년 / 10년`
- 기본 기간: `10년`
- 월 적립금 프리셋: `30만 / 50만 / 100만 / 200만 / 300만`
- 기본 적립금 프리셋: `100만원`
- 사용자가 직접 바꿀 수 있는 공통 입력은 `기간`, `월 적립금`, `자산 선택`이다.
- 포트폴리오 비중 변경은 rev1에서 직접 편집 없이 `preset only`

## 14. 레거시 처리 방침
### 14.1. 현재 구조 재해석
- 현재 라우트 구조는 `/`, `/prices`, `/correlation`, `/indicators`, `/strategy` 기준이다.
- rev1에서는 이 구조를 그대로 노출하는 것이 목적이 아니다.
- `적립식 메인 1개 + 지표 상세 보조 구조`로 재편한다.

### 14.2. 페이지별 처리
- `DashboardPage`: rev1 메인 메뉴에서 노출하지 않으며, Silver에서는 리디렉션 또는 차단 대상으로 본다.
- `PricePage`: rev1 메인 메뉴에서 노출하지 않으며, Silver에서는 리디렉션 또는 차단 대상으로 본다.
- `CorrelationPage`: rev1 메인 메뉴에서 노출하지 않으며, Silver에서는 리디렉션 또는 차단 대상으로 본다.
- `IndicatorSignalPage`: 삭제하지 않고 기능 변경 후 재구성 대상으로 유지한다.
- `StrategyPage`: rev1 기준 완전 drop 대상으로 본다.

### 14.3. 재사용 가능 요소
- `PricePage`의 다중 자산 가격/정규화 수익률 비교 로직은 탭 A 차트 로직 설계 참고용으로 재사용 가능하다.
- `IndicatorSignalPage`의 RSI/MACD/ATR 관련 기능은 요약 카드 및 보조 상세 페이지 기반으로 재사용 가능하다.
- 기존 `StrategyPage`의 MACD/RSI/ATR 전략 백테스트는 rev1 조건 전략 비교와 개념이 다르므로 직접 계승하지 않는다.

## 15. 주요 컴포넌트 요구사항
### 15.1. 메인 적립식 비교 화면
- 상단 입력 패널
- 탭 네비게이션
- 핵심 비교 차트
- 요약 KPI 카드
- 위험 설명 카드
- 조건/가정 설명 카드
- 필요 시 지표 요약 카드

### 15.2. 차트/카드 최소 요구사항
- 차트는 비교 대상의 차이를 한눈에 봐야 한다.
- 카드 문구는 초보 투자자 기준으로 해석 가능해야 한다.
- 수치만 나열하지 말고 의미를 붙여야 한다.

## 16. 데이터 및 인터페이스 전제
### 16.1. 데이터 전제
- rev1 구현 시 신규 미국 자산군은 수집 파이프라인 확장이 필요하다.
- 문서 단계에서는 `지원 예정 자산`으로 정의하되, 이미 구현되어 있다고 가정하지 않는다.

### 16.2. 계산 전제
- replay 시뮬레이션
- 배당/이자 재투자 고정
- 매 날짜 환율 기준 환산
- 세금/수수료/슬리피지 제외
- 소수 단위 매수 허용

### 16.3. API/화면 인터페이스 전제
- 메인 화면은 탭 단위로 구성한다.
- 각 탭은 공통 입력을 공유하되, 자산 선택 범위와 preset은 탭별로 다를 수 있다.
- 보조 상세 페이지는 메인 결과 해석을 방해하지 않는 선에서만 연결한다.

## 17. 검증 기준
### 17.1. 제품 검증
- 초보 투자자가 메인 화면만으로 비교 축을 이해할 수 있어야 한다.
- 첫 진입 후 `무엇을 비교하는 화면인지`가 즉시 보여야 한다.
- `적립식 기준선`이 모든 비교의 중심으로 유지돼야 한다.

### 17.2. 구현 검증
- 탭 A, B, C가 모두 동작해야 한다.
- 자산/전략/preset 제한 규칙이 UI와 계산 로직에서 일치해야 한다.
- `WBI`가 실자산처럼 수집 로직을 타지 않아야 한다.
- 레거시 페이지 처리 방침이 문서와 실제 메뉴 구조에서 일치해야 한다.
- `Dashboard / Price / Correlation`은 Silver에서 직접 접근 시 리디렉션 또는 차단 정책이 일관되게 적용되어야 한다.

## 18. 리스크
- 미국 자산군 수집 경로 확장이 선행되지 않으면 rev1 핵심 자산 universe를 완전 지원할 수 없다.
- `연도 기준 최대 손실폭` 정의를 계산 로직과 UX 문구에서 다르게 쓰면 사용자 혼선이 생긴다.
- 조건 전략 비교를 기존 StrategyPage 개념과 혼동하면 IA가 다시 복잡해질 수 있다.
- 디자인 레퍼런스를 IA까지 복제하면 rev1의 제품 목적과 어긋난다.
- 복잡한 탭의 레이아웃을 카드형으로 유지할지 step형 예외를 둘지는 실제 시안 검토 후 결정해야 한다.
- Bronze에서 Silver로의 교체 배포 방식은 코드 작성 전 별도 운영 계획 검토가 필요하다.

## 19. 구현 우선순위
1. 메인 메뉴 구조 확정
2. 탭 A/B/C 정보 구조 확정
3. 탭별 기본값 / 허용 자산 / preset 고정
4. 주요 차트 / KPI 카드 / 설명 카드 정의
5. 데이터 확장 필요 자산과 synthetic 자산 처리 규칙 반영
6. 레거시 페이지 연결 정책 반영

## 20. 요약 결정문
- Silver rev1은 `적립식 비교` 중심 제품이다.
- 메인 구조는 3탭 비교형으로 고정한다.
- `Dashboard`, `Price`, `Correlation`은 rev1 메인 노출 대상이 아니다.
- `Strategy`는 drop 대상이다.
- `IndicatorSignal`은 보조 상세 구조로 재구성한다.
- `WBI`는 synthetic benchmark다.
- 미국 ETF/주식 자산군은 신규 데이터 확장 전제로 구현 계획을 세운다.
- 탭 A는 기본 3개 자산을 먼저 노출한다.
- 공통 입력은 기간, 월 적립금, 자산 선택으로 제한한다.
- 환율은 매 날짜 환율 기준으로 계산한다.
- 탭 C는 실제 리밸런싱 매도/매수와 소수 단위 매수를 허용한다.
