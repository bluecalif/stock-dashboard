### 2-7. 컨텍스트 엔지니어링 + 메모리

- 목적
    - 왜 컨텍스트가 필요한가? → 관심이 많은 질문/조회 내용에 대해 미리 준비. 최종 목표가 있다면 현재까지 지식의 갭을 알고 있는 것이 좋음
    - 
- 대화 요약
    - 항목별 요약: 페이지별 및 아이템 결과 요약
    - 시간별 요약: 항목별 요약에 타임스탬프만 붙이기
    - 기타  관심 영역 레코딩
- 메타 데이터
    - 페이지/주차별 방문회수, 아이템/주차별 조회수
    - 자산별 조회수
- 운영 방식
    - 요약 빈도 및 저장 형태: 5번에 한번 저장. 위의 요약 및 메타데이터 저장
    - 컨텍스트 오프로딩/온로딩 규칙:
        - 질문에 대해 위의 저장된 내용을 어떻게 검색 및 호출할 것인지 결정. 벡터DB까지 필요한지 검토 필요
        - 나중에 온보딩에 사용될 내용 생성을 위한 컨텍스트 온로딩 규칙 검토

### 2-8. UX 온보딩

- 전제
    - 앱의 시작부터 끝까지 사용자를 가이드할 것
    - 사용자 유형별로 구분
        - Beginner, Expert, Feeling, Logic
- UI 형태
    - 로그인 및 세션 시작 온보딩 + 지속적인 사용자 가이드
    - 별도의 창이나 페이지로 처리할지, 기존 Component에 얹어서 만들지
- 시나리오 Flow
    - 시작 온보딩
        - Introduction:  앱의 목적 설명 (사용자의 합리적 투자 결정 Companion 등등)
        - How to Use: 페이지로 직접 이동 vs 채팅에서 문의
        - Ice Breaking ( two questions): 사용자 Type 가늠하는 질문
    - 지속적인 가이드 (CG: Continuous Guide)
        - 페이지 분기: 채팅 질문 Navigation (기구현) + Navigation 이전에 사전 설명 (본 페이지의 설명 및 사용자의 요청이 다른 페이지에 있으므로 그쪽으로 이동한다는 설명)
        - Ice Breaking 기반 사용자 페르소나 기반 페이지/아이템 추천
        - 이력 기반 페이지/아이템 추천
        - Out of Bound 처리: 본 앱에 없는 기능 요구하는 경우 대안 제시 (웹 사이트 추천등)
    - Future Action & Closing: 리포트 요약 및 향후 제안
